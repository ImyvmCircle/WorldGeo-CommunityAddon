# 用户侧文档规则

## 叙述视角

1. 面向玩家、访客、普通使用者或类似对象的说明文档只使用第三人称。
2. 不使用第一人称或第二人称，包括"我""我们""你""你的""你们"等称呼。
3. 使用明确主体描述对象、权限与行为，例如"玩家""访客""成员""所有者""管理员""申请者""受邀者""服务器管理员"等。
4. 命令名、按钮名、菜单名、页面名和界面原文可以按实际文本书写；解释机制时仍保持第三人称。

## 内容边界

1. 只简介陈述项目内容和确定的使用方法。
2. 只讲事实和 prompt 要求的内容。
3. 不写针对项目的建议。
4. 不加入 prompt 没有提及的应用语境假设。
5. 不加入 prompt 没有提及的防御性、补丁性或免责性内容。

## 文风检查

1. 生成前遵守 `agents/WRITING_STYLE.md`。
2. 生成后按 `agents/WRITING_STYLE.md` 排查并合理去除 AI 文字痕迹。
3. 语言保持有人味的自然描述，但不牺牲机制事实、操作对象和使用条件的准确性。
