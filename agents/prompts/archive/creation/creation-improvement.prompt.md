name: community-creation-improvement
description: 调整聚落创建确认流程与提示
---

现在更新创建逻辑。请遵循`agents/instructions/development.instructions.md`所提及的各种原则

1. 命令创建聚落申请之时，需要有个可交互的确认，该交互提供与目前用Menu创建时生成的`ConfirmMenu`一致的信息提醒玩家。如确认，再执行。不然，无事发生。
2. 把Menu创建聚落之时的`ConfirmMenu`取消，窗口点击创建时，不再发送`ConfirmMenu`，而是收到上面这条确认消息
3. 确认消息增加：
   - 第一条增加聚落的名字、类型和形状（后两个用人类可读的语言文件而不是原类型名称显示）
   - 第二条增加面积附加费用，首先是该聚落的面积附加费用，然后是计算方法。对于`MANOR`，每1万平米收费1000元，对于`REALM`，每4万平米收费3000元。第一个计价单位免费。不满一个计价单位的面积按相应价格*小数单位收费即可，不取整。
4. 因计价消息改动，创建聚落面板中的错误提示和条件也要相应改动。
5. 因计价方式改动，扣费方式也要改动，扣费时的相应发给玩家的消息也要改动，退还费用的逻辑和相应发给玩家的消息也要进行改动。
6. 一切发给玩家的消息都需要按照`agents/instructions/development.instructions.md`第一条之要求进行改进。
