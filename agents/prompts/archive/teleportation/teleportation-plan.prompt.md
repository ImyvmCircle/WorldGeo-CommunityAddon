# 传送系统更改方案

第一位是要遵循`agents/instructions/development.instructions.md`。现在，前置修复工作1.已经完成，不用再管，请完成剩下的所有工作。

## 任务上下文

1. Community通过regionNumberId连接到附带的有且仅有一个的Region
2. Region的传送点是跟其所属的一个Scope集合中的每个Scope一对一绑定的。每个传送点有一个坐标位置（可以为null），有一个可达性开关。
3. Region的主传送点是其Scope集合中的第一个传送点。
4. 新建Region时候一定会附带一个main_scope. 这个main_scope的传送点，因其位于Scope集合的第一个位置，故而就是Region的总传送点。

## 前置修复工作

1. Accessibility:
   - 目前，通过Menu进行传送点的Accessibility的toggle时，语言文件中未能体现出状态的变化.
   - 对其本意的执行也缺乏。这个参数本意上是一个作为检查工具的参数，检查的不是传送点是否可用，而是传送点是否公开。那么，对应聚落，就是，除了该聚落的正式成员之外的其他玩家是否可以使用这个传送点。
     - 那么，相对应的，对于现在已有的、公开的聚落列表里的一个对应聚落打开时候，应该要新增一个传送按钮，能够打开这个聚落的传送GeoScope列表进行传送。传送时，要对能否传送进行检查
       - 关于这个传送列表，也有一个需要进行修复的地方。其总按钮Global的含义明显偏离主传送点的含义。这是因为其跟Geometry Modification 和 Setting等共用菜单导致的。
2. 通过本mod中已有的机制创建之后的Scope之时，自动通过`ImyvmWorldGeo`相关接口将传送点设置为null, accessibility设置为false；创建聚落时候，因会自动创建一个附带的main_scope，所以直接放着不动就好。

## 大体内容

### 成本机制

#### 传送点设置收费

传送点设立需要增加收费机制，对一个区域内的第一个传送点的设立可以依据创建聚落时候创建区域的附带免费赠送，但是后面的传送点价格要翻倍式增高到无法负担水平。实现机制如下：

1. 设定传送点的坐标时：增加一项检查，即聚落已有的非null传送点数量。这一检查应该是通过ImyvmWorldGeo的Ap实现的。设立第二个有效传送点时收费2000，然后每新增一个，价格翻一倍
2. 修改非null传送点的时候，固定收取3000.

#### 传送延时和单次收费

传送点使用需要增加延时机制，且延时按一日之内使用同一个聚落的任意传送点次数计算：

1. 对于作为聚落正式成员的玩家，头10次使用免费
2. 对于非正式成员的玩家，免费次数只有1次
3. 传送免费次数用完之后的第一次传送从10块钱和2秒开始收取费用和传送延时，同样也是指数翻倍。传送延时结束前移动或者被攻击，则传送无效。传送时的玩家本人余额不足时，传送同样取消。

#### 传送信息提示

传送信息是通过Text聊天栏文本或邮件系统发出的，提示意在告知操作的完成和允许玩家对传送中所耗费成本进行确认。显示以下几种公告：
1. 设定传送点时。告知设定的传送点的所在Region成本及缘由（如果已有传送点，还要警告），并设置可交互信息选项。这一项一定要跟GUIDELINES中的PendingOperation机制结合起来，允许玩家在一定时间内点击确认设定传送点并支付费用，或者取消设定传送点。
2. 设定传送点完成时，给聚落所有正式成员（参考GUIDELINES）发邮件通知完整的设定情况。
3. 开始传送时，要给被传送玩家提示目的地，花费和时延，并且提示如果移动或者被攻击就会取消传送；取消传送时也要发送对应消息，说明取消原因。
4. 传送完成也要给被传送的玩家发送消息。

#### 传送效果

传送后，给被传送的玩家一个非常短暂的晕眩效果和周围地面上重锤打击的粒子效果。
