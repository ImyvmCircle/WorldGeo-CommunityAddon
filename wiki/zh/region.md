# 辖区

每一个聚落，都占据着这片大地上的一块疆域。这块疆域被称为**地块**（Region）——它是聚落存在于世界中的地理根基，是聚落宣示主权、设定规则、划定边界的空间载体。

然而，一个聚落的版图往往不是一块规整的连续土地。它可能由多片分散的区域拼合而成——祖地、新拓之地、飞地、特区——每一片都有各自的形状、职能与规则。这些构成地块的每一片独立区域，我们称之为**辖区范围**（Geoscope）。

**Geoscope 是 Region 的组成单元。** 一个 Region 的实际疆域，正是其所有 Geoscope 覆盖范围的集合。在世界地图上，你看到的那片属于某聚落的土地，就是这个聚落所有 Geoscope 叠加起来的轮廓。

---

## 一个聚落的疆域是如何构成的

聚落建立之初，系统会自动为其创建第一个 Geoscope，也就是聚落的**主辖区**。这片土地是聚落最初的根基。

随着聚落发展，你可以**增加新的 Geoscope** 来拓展版图——无论是紧邻主辖区的延伸，还是隔着一段距离的分散飞地。每个 Geoscope 都可以独立设定形状（矩形、圆形或多边形），以贴合地形或建筑规划。

你也可以随时**修改某个 Geoscope 的边界**，重新划定那片区域的范围。

所有这些 Geoscope 共同构成这个聚落的 Region，也就是聚落在这个世界中实际掌控的土地。

---

## 圈地模式

要定义或修改一个 Geoscope 的边界，需要使用**圈地模式**在世界中亲手选取点位。

- **进入圈地模式**后，手持**命令方块**，在世界中**右键点击方块**即可设置选择点位。
- 系统支持三种圈地形状：
  - **矩形**：选择两个对角顶点，自动生成矩形区域。
  - **圆形**：先选择圆心，再选择圆边上的一点，以确定半径。
  - **多边形**：依次选择所有顶点，围成自定义多边形（须为凸多边形）。
- 不必一次性完成所有点位的选择，可以退出菜单后随时继续。
- 建议用方块在地面标记边界点，方便日后参考。

> **提示：** 圈地模式有两种形态，对应不同的操作——**新建圈地模式**用于创建新的 Geoscope，**修改圈地模式**用于调整已有 Geoscope 的边界。两者不能同时启用。如果你打开"修改地块"时收到警告，说明你当前处于新建圈地模式，请先在辖区创建菜单中关闭圈地模式，再来修改地块。

## 增加辖区范围

要为聚落添加一个新的 Geoscope，可通过以下两个入口：

- **圈地菜单**（主菜单 → **圈地**按钮）→ **增加辖区**；或
- 内政管理菜单 → **地理范围修改** → **全局** → **添加辖区范围**。

两种方式均会打开**辖区创建界面**。在这里，你可以：

1. **开启圈地模式**，进入世界中选择边界点位。
   - 菜单标题会提示当前步骤（"先开启圈地模式" → "在世界中选择点位"）。
   - 点位充足后，会出现**命名**按钮和**确认创建**按钮。
2. **切换形状**（矩形 / 圆形 / 多边形）。
3. 选完点位后**退出菜单**，在世界中继续选点；或直接**关闭圈地模式**（取消本次操作）。
4. 若想清空已选点位重新来过，点击**重置选点**按钮。
5. 凑够点位并命名后，点击**确认创建**，系统生成费用摘要并等待你在聊天框确认。

创建一个新 Geoscope 的费用：

| 聚落类型 | 新增辖区基础费用 |
|---|---|
| 庄园 | $500.00 |
| 领地 | $1,000.00 |

此外，还需按 Geoscope 所覆盖的面积额外计费。

**软性上限机制：** 推荐的 Geoscope 数量上限为 `floor(正式成员数 / 2)`（正式成员指角色为所有者、行政助理或成员者）。超过此上限**不会阻止**你继续创建，但**固定费用**将随超额数量递增：每超出上限 1 个，固定费用乘以 **1.5**（150%）。例如上限为 2 个，若已有 2 个 Geoscope 再创建第 3 个（超出 1 个），基础费用上调至 150%；若已有 3 个再创建第 4 个（超出 2 个），则上调至 225%。面积计费和退款不受此机制影响。超出上限时，系统会在确认费用前发出提示，告知你当前溢价比例。

> **注意：** 只有处于**激活状态**的聚落才能添加辖区范围。招募中或待审核阶段的聚落尚不允许此操作，需等待通过 OP 审核进入激活状态后方可进行。

**对应命令：**
- `/community confirm_modification <regionId> <scopeName>` — 确认创建
- `/community cancel_modification <regionId> <scopeName>` — 取消创建

## 修改 Geoscope 边界

你可以重新划定某个已有 Geoscope 的边界范围。修改前，系统会计算新旧面积的差值并相应收费或退款。

**通过圈地菜单（推荐）：**

1. 主菜单 → **圈地** → **修改地块**；
2. 若你尚未进入任何圈地模式，选择聚落（如加入了多个），再从列表中点击要修改的辖区范围；
3. 系统自动进入**修改圈地模式**，菜单关闭，提示你在世界中选取新的边界点位；
4. 手持命令方块，右键点击方块设定新边界；
5. 再次打开圈地菜单 → **修改地块**，确认修改；
6. 聊天框出现费用摘要及**[确认]** / **[取消]**提示（5 分钟内有效）。

**通过内政管理菜单：**

1. 内政管理菜单 → **地理范围修改** → 从列表选择要修改的辖区范围；
2. 进入修改圈地模式，菜单关闭并提示；
3. 按上述第 4–6 步操作。

**费用规则：**
- 扩大辖区面积：支付新增面积的差价。
- 缩小辖区面积：退还缩减面积差价的 **50%**。

> **注意：** 只有处于**激活状态**的聚落才能修改 Geoscope 边界。招募中或待审核阶段的聚落尚不允许此操作。

> **注意：** 需要**地理范围修改**行政授权。

## 地块设置项修改

每个 Geoscope 可以独立配置**权限**与**规则**，精细控制其他玩家在该区域内的行为。

进入方式：内政管理菜单 → **地块设置项修改** → 选择要配置的辖区范围 → **设置菜单**。

### 权限

权限控制非所有者玩家在辖区范围内可以进行的操作。每项权限可在整个辖区范围内全局切换，也可按玩家单独分配。

| 权限名称 | 说明 |
|---|---|
| **建造/破坏** | 放置和破坏方块 |
| **建造** | 仅放置方块 |
| **破坏** | 仅破坏方块 |
| **访问容器** | 打开箱子、木桶、潜影盒等 |
| **交互** | 使用按钮、拉杆、门等 |
| **红石使用** | 与红石组件交互 |
| **村民交易** | 与村民交易 |
| **放置桶** | 用桶放置液体 |
| **取水桶** | 用桶舀取液体 |
| **耕作** | 耕耘土地、收割作物 |
| **PvP战斗** | 对其他玩家造成伤害 |
| **击杀动物** | 伤害被动动物 |
| **击杀村民** | 伤害村民 |
| **使用投掷物** | 投掷物品（雪球、鸡蛋、药水等） |
| **投掷鸡蛋** | 投掷鸡蛋 |
| **投掷雪球** | 投掷雪球 |
| **投掷药水** | 投掷 / 使用药水 |
| **点火** | 使用打火石或火焰弹 |
| **使用盔甲架** | 与盔甲架交互 |
| **使用物品展示框** | 与物品展示框交互 |

每项权限启用时按面积计费，恢复默认退还该费用的 **50%**。按玩家单独设置的费用为全局费用的 **1/5**。

### 规则

规则控制辖区范围内的自动行为（不与特定玩家操作绑定）。

| 规则名称 | 说明 |
|---|---|
| **怪物生成** | 允许敌对生物在辖区内生成 |
| **幻翼生成** | 允许幻翼在辖区内生成 |
| **TNT方块保护** | 防止 TNT 破坏辖区内方块 |

规则启用时同样按面积计费。

> **注意：** 需要**地块设置项修改**行政授权。

**对应命令：**
- `/community confirm_setting <regionId>` — 确认设置修改
- `/community cancel_setting <regionId>` — 取消设置修改
